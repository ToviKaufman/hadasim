 
# Trip Tracking System

## תיאור הפרויקט
מערכת לניהול ומעקב אחר טיול תלמידות בזמן אמת.

המערכת מיועדת לשלושה סוגי משתמשים:
תלמידות, מורות והורים.

המערכת מאפשרת:
- רישום לטיול
- צפייה בפרטי משתמשים
- מעקב בזמן אמת אחר מיקום תלמידות על גבי מפה
- התרעות על התרחקות מהקבוצה , תלמידה שרחוקה מידי האייקון שלה צבוע באדום.

---

## מטרות המערכת
- רישום וניהול משתתפות בטיול  
- מעקב בזמן אמת אחר מיקום תלמידות  
- מתן שליטה ובקרה למורות  
- שקיפות להורים לגבי מיקום התלמידות  

---

## טכנולוגיות

### Server
- ASP.NET Core Web API  
- SignalR (עדכונים בזמן אמת)  
- SQL Server  

### Client
- React  
- Leaflet (מפות)  
- Context API (ניהול state של מיקומים ומשתמשים)  

---

## בחירות טכנולוגיות
- SignalR מאפשר עדכונים בזמן אמת ללא polling  
- React Context מאפשר שיתוף state בין קומפוננטות בצורה פשוטה  
- ארכיטקטורת שכבות (API / BL / DAL) מפרידה בין לוגיקה, נתונים ו־API  
- Leaflet מאפשרת הצגת מפות ומיקומים בצורה נוחה  

---

## פיצ'רים עיקריים

### תלמידות
- הרשמה לטיול  
- עדכון מיקום באמצעות סימולטור

---

## סימולציית מיקומים
המערכת כוללת סימולטור המדמה GPS:

- עדכון מיקום כל 2 שניות  
- יצירת מיקום רנדומלי בטווח הגיוני  
- שליחת נתונים לשרת  
- הפצה לכל הלקוחות באמצעות SignalR  
- עדכון ממשק משתמש באמצעות Context  

---

## ------------------------------- Flow המערכת -----------------------------  
1. התחלת סימולצית מיקום
2. עדכון מיקום נשלח לשרת
3. הפצת עדכון לקליינט ע"י SignalR
4. הקליינט מעדכן UI דרך Context
5. משתמש מזין תעודת זהות
6. בדיקה האם המשתמש קיים:
7. אם לא → רישום
8. אם כן → טעינת פרטים
9. אם המשתמש מורה - הצגת דשבורד מורה


---

## ארכיטקטורה

###  Server
- API Layer – חשיפת endpoints  
- BL – לוגיקה עסקית  
- DAL – גישה למסד נתונים  

### Client
- מבוסס קומפוננטות React  
- שימוש ב־Context לניהול state גלובלי  
- הפרדה בין דפים, שירותים ולוגיקה

## הוראות הרצה

### Clone
git clone <repository-url>

### Database (SQL Server)

קובץ:
appsettings.json

Connection String:
"ConnectionStrings": {
  "DefaultConnection": "Server=YOUR_SERVER;Database=TripDB;Trusted_Connection=True;"
}
הרצת מיגרציות:
Update-Database  
  


### Server
- לפתוח את ה־Solution ב־Visual Studio  
- להגדיר את פרויקט ה־API כ־Startup Project  
- להריץ את הפרויקט  


### Client
cd client  
npm install  
npm start
